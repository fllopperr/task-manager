# 6. Оценка проекта

Анализ проводится по двум основным направлениям:

- оценка проекта по ключевым критериям качества программного продукта;
- сравнение с существующими решениями на рынке task management систем.

---

# 6.1 По каким критериям можно оценить проект

Для оценки качества системы используются несколько групп критериев.

## Функциональная полнота

Оценивается соответствие реализованной функциональности заявленным требованиям:

- создание и редактирование задач;
- управление досками;
- организация задач по колонкам (Kanban);
- фильтрация и поиск задач;
- управление пользователями;
- поддержка совместной работы.

Функциональная полнота показывает, насколько система покрывает реальные сценарии использования.

---

## Производительность

Производительность является критически важным параметром для систем управления задачами, поскольку пользователи часто выполняют большое количество операций:

- перемещение задач между колонками;
- редактирование задач;
- обновление данных в режиме реального времени.

Основные показатели:

- время ответа API;
- скорость рендеринга интерфейса;
- устойчивость системы под нагрузкой.

По данным исследований рынка программ управления задачами, **пользователи ожидают время отклика интерфейса менее 300 мс**, иначе снижается воспринимаемая скорость работы приложения.

Источник:
https://www.nngroup.com/articles/response-times-3-important-limits/

---

## Масштабируемость

Масштабируемость показывает способность системы обслуживать растущее количество пользователей без значительного снижения производительности.

В рамках проекта учитываются:

- возможность горизонтального масштабирования backend;
- использование Redis для кеширования;
- использование GraphQL для эффективной передачи данных.

---

## Удобство пользовательского интерфейса (UX)

User Experience является одним из ключевых факторов успешности task-менеджеров.

Основные критерии:

- простота интерфейса;
- минимальное количество действий для создания задачи;
- визуальная понятность Kanban-доски;
- адаптивность интерфейса.

По данным исследований UX-индустрии, **88% пользователей не возвращаются на сайт после плохого пользовательского опыта**.

Источник:
https://sweor.com/firstimpressions

---

## Надежность

Надежность системы определяется:

- стабильностью работы сервера;
- корректной обработкой ошибок;
- сохранностью пользовательских данных.

В рамках проекта используются:

- транзакции базы данных;
- валидация данных;
- централизованная обработка ошибок.

---

## Безопасность

Безопасность оценивается по следующим параметрам:

- защита пользовательских данных;
- безопасная аутентификация;
- защита от распространённых атак.

В проекте используются:

- JWT-аутентификация;
- хеширование паролей;
- HTTPS соединения.

---

# 6.2 Сравнение с конкурентами

Для оценки конкурентоспособности проекта было проведено сравнение TaskFlow с популярными системами управления задачами.

Основные конкуренты:

- Trello
- Asana
- ClickUp
- Notion

---

## Trello

Trello является одним из самых популярных Kanban-инструментов для управления задачами.

Компания Atlassian сообщает, что **Trello используется более чем 50 миллионами пользователей по всему миру**.

Источник:
https://www.atlassian.com/software/trello

Преимущества:

- простой интерфейс;
- удобная Kanban-система;
- высокая популярность.

Недостатки:

- ограниченная функциональность бесплатной версии;
- слабая кастомизация.

---

## Asana

Asana — мощная система управления проектами, ориентированная на команды и корпоративный сегмент.

По данным компании, **Asana используют более 100 000 организаций по всему миру**.

Источник:
https://asana.com

Преимущества:

- мощные инструменты управления проектами;
- интеграции;
- аналитика.

Недостатки:

- высокая сложность интерфейса;
- избыточность для небольших команд.

---

## ClickUp

ClickUp — современная платформа управления задачами с большим количеством функций.

Согласно официальным данным, **ClickUp используют более 800 000 команд**.

Источник:
https://clickup.com

Преимущества:

- высокая функциональность;
- гибкая настройка интерфейса.

Недостатки:

- сложность освоения;
- перегруженный интерфейс.

---

## Notion

Notion представляет собой универсальный инструмент для организации информации, заметок и задач.

По данным компании, **Notion используют более 30 миллионов пользователей**.

Источник:
https://www.notion.so

Преимущества:

- высокая гибкость;
- возможность создания собственных систем управления задачами.

Недостатки:

- не специализирован для Kanban;
- требует настройки пользователем.

# Итог оценки

Проект TaskFlow ориентирован на сегмент пользователей, которым требуется **простой, быстрый и минималистичный инструмент управления задачами**.

По сравнению с существующими решениями:

- TaskFlow выигрывает в **простоте интерфейса и скорости работы**;
- уступает крупным системам в **глубине функциональности**;
- обладает потенциалом для развития и расширения возможностей.

Таким образом, TaskFlow может успешно занять нишу **легковесных task-менеджеров для индивидуальных пользователей и небольших команд**.
